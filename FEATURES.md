# tmx 功能演示

## 🎬 自动启动 tmux 功能

当你运行 `tmx` 时，如果 tmux 未运行，会出现这个提示：

```
📝 tmux 未运行

💡 tmx 可以自动启动 tmux 并创建默认会话
是否自动启动? [Y/n]:
```

### 交互流程

1. **按 Enter 或输入 Y**
   - 自动启动 tmux
   - 创建名为 "default" 的会话
   - 会话窗口名称为 "tmx"
   - 自动打开会话管理器

2. **输入 n**
   - 显示手动启动 tmux 的提示
   - 退出程序

### 优势

✅ **更友好** - 不需要记住先启动 tmux
✅ **更便捷** - 一个命令完成所有操作
✅ **更智能** - 自动创建默认会话
✅ **可选** - 用户可以选择不自动启动

## 📝 使用场景对比

### 之前的流程

```bash
# 1. 需要先启动 tmux
$ tmux

# 2. 然后运行 tmx
$ tmx

# 3. 如果忘记启动 tmux，会报错
$ tmx
错误: tmux 未运行
```

### 现在的流程

```bash
# 直接运行 tmx
$ tmx
📝 tmux 未运行

💡 tmx 可以自动启动 tmux 并创建默认会话
是否自动启动? [Y/n]: [按 Enter]

🚀 正在启动 tmux...
[自动进入 tmux 并打开管理器]
```

## 🎯 完整使用流程

### 第一次使用

```bash
$ tmx
📝 tmux 未运行

💡 tmx 可以自动启动 tmux 并创建默认会话
是否自动启动? [Y/n]: [Enter]

🚀 正在启动 tmux...
[进入 tmux 和会话管理器]
```

### 安装配置

```bash
# 按 Ctrl+b t 打开管理器（如果已配置）
# 或在 tmux 中运行
$ tmx --install
✓ 已添加配置到: /Users/xxx/.tmux.conf

请运行以下命令重新加载 tmux 配置：
  tmux source-file ~/.tmux.conf
```

### 重新加载配置

```bash
$ tmux source-file ~/.tmux.conf
```

### 使用快捷键

现在状态栏会显示：
```
[Ctrl+B T] 管理器 | 14:30 2025-01-23
```

按 `Ctrl+b t` 就能随时打开管理器！
